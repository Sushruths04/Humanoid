# P1.4 — SeqNav (Humanoid-G1-SeqNav-v0)

## Summary

The robot must visit **two markers in a specific order**: first A, then B (e.g., "go to red, then blue"). The order must be correct.

**Result: 80.9% full-sequence success, 94.5% ordering accuracy, 97.7% first-subgoal rate.**

This was the hardest checkpoint — multiple failed attempts before the fix. Read [[SeqNav Stand-Still Local Optimum]] for the full story.

[Full result doc](../../docs/results/humanoid-g1-seqnav-v0.md)

## The Architecture (after the rebuild + bootstrap fix)

Built entirely on the proven CommandNav core. Only addition: a `seq_advance` event.

```python
# Reset: sample ordered subgoal sequence, point command at subgoal-0
def reset_seq_nav(env, env_ids, ...):
    _ensure_buffers(env, num_markers)          # CommandNav buffers
    _ensure_seq_buffers(env, num_subgoals)     # just _seq_targets + _seq_phase
    env._nav_markers_xy[env_ids] = ...        # sample positions
    env._seq_targets[env_ids] = sample_subgoal_sequence(...)  # ordered marker ids
    env._seq_phase[env_ids] = 0
    env._nav_target_ids[env_ids] = env._seq_targets[env_ids, 0]  # point at subgoal-0

# Per-step: advance phase when subgoal reached, re-point command
def seq_advance(env, env_ids, ...):
    dist = (robot_xy - commanded_target_xy(env)).norm(dim=-1)
    new_phase, advanced = advance_subgoal(env._seq_phase, dist, reach_radius, num_subgoals)
    env._seq_phase = new_phase
    if advanced.any():
        env._nav_target_ids[advanced] = env._seq_targets[arange, env._seq_phase][advanced]
        env._nav_prev_xy[advanced] = robot_xy[advanced]   # reset progress baseline!

# Steer: SAME as CommandNav (steer_velocity_to_target)
# Obs: SAME as CommandNav (one-hot + relative vector to current target)
# Reward: SAME as CommandNav (nav_command_reward, progress_scale=2.0, wrong_penalty=1.0)
```

**Event order matters:** `seq_advance` must be registered BEFORE `steer_velocity` so the steer uses the freshly-advanced target in the same step.
