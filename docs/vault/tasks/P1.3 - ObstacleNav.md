# P1.3 — ObstacleNav (Humanoid-G1-ObstacleNav-v0)

## Summary

CommandNav plus 3 randomly placed cylindrical obstacles. The robot must navigate to the commanded target while avoiding obstacles (no collisions).

**Result: 85.9% goal success** (per-command: [83.7%, 88.0%]), collision penalty at end of training ≈ −0.0008 (negligible).

[Full result doc](../../docs/results/humanoid-g1-obstaclenav-v0.md)

---

## What's Different from CommandNav

1. **Reset event** also places random obstacles (`sample_marker_positions` reused for obstacles at a different radius range)
2. **Steer event** uses `velocity_command_to_target_avoiding` (potential-field repulsion from obstacles)
3. **Extra observation**: relative positions of all obstacles (6-dim = 3 obstacles × 2D)
4. **Extra reward term**: `collision_penalty` (smooth proximity penalty)

The nav reward (`nav_command`) reuses `nav_command_reward` from CommandNav verbatim.

---

## Why It Works Despite Obstacles

The potential-field steering in `velocity_command_to_target_avoiding` smoothly pushes the command sideways around obstacles. The policy doesn't need to learn to avoid obstacles — the steering law handles it. The policy just needs to learn to track the steering command (which it already does from CommandNav training).
