Language & Vision (the VLA)

"VLA" = Vision-Language-Action: a robot that sees, understands an instruction, and acts. Here's how far ours actually goes — and where we draw the honesty line.

Language: from "fake" to "on"

The first version fed the policy a fixed dummy code for the instruction — so the robot literally couldn't tell commands apart. That's honest wiring, not a working language model. We're flipping it on with two changes:

  1. Randomize the command each episode (e.g. walk forward, turn left, stand still, walk fast…).
  2. Add a reward that depends on the command — track the target speed/turn it maps to. This is the actual on-switch: now the robot has a reason and a way to listen.
Already verified (on CPU): distinct commands → distinct velocity targets → the tracking reward separates them cleanly (e.g. "stand still" → reward peaks at zero speed, "fast" > "slow" > "stand"). On GPU we train it and produce a behavior-separation plot as proof.

Vision: the camera, and the bug we beat

We mount a camera on the G1's head, normalize the pixels, and feed a small CNN (the classic 3-layer "Nature-CNN") shared by the actor and critic. The hard part wasn't the network — it was a graphics driver bug:

The Vulkan blocker (solved). Camera rendering needs Vulkan; the container could compute (CUDA worked) but couldn't render (libGLX_nvidia.so.0 missing). Fix: run headless with Isaac Lab's rendering kit + --enable_cameras at low resolution. Camera training then reached PPO and scaled to 2048 envs. Lesson: "nvidia-smi works" ≠ "rendering works" — compute and graphics are provisioned separately.

Honest status: the vision pipeline runs, but only at smoke scale — no long vision run or saved vision policy yet.

Interview answer — "is it really a VLA?"
"It's a VLA-shaped scaffold and I'm precise about it: Action is real benchmarked PPO; Vision is a real camera+CNN pipeline that runs but is only smoke-tested; Language was a fixed placeholder embedding that I'm turning on by randomizing the command per episode and adding a command-tracking reward, with a behavior-separation plot as the acceptance test. The clean interface means I can swap the hash embedding for a frozen CLIP encoder without touching the env."
