The Tech Stack

Every tool we use, what it does, and where it sits — bottom (hardware) to top (your task).

What each piece does

ToolWhat it isWhy we use it
Isaac SimNVIDIA's GPU-accelerated physics simulatorRuns thousands of robot copies at once — that speed is the method
Isaac LabRL framework on top of Isaac SimReady-made G1 robot + a clean way to compose observations & rewards
RSL-RLA fast PPO implementationThe standard trainer for legged robots; GPU-vectorized
PPOThe RL algorithm (Proximal Policy Optimization)Stable, simple, perfect when simulation is cheap
GR00TNVIDIA's humanoid foundation model (~3B params)Pre-trained brain we fine-tune for manipulation
AMPAdversarial Motion Priors (a "style" reward)Makes motions look natural without hand-tuning every joint
DockerA portable container of the whole environmentPull one image → identical setup on any machine
ModalServerless GPU cloudFire-and-forget jobs, pay per second, parallel
Lightning AIRented persistent GPU boxInteractive debugging + anything needing rendering
Hugging FaceCloud storage for models/dataWhere big checkpoints live (kept out of GitHub)
One non-obvious constraint. Isaac Sim needs a GPU with RT cores — so L40S / L4 / A10 work, but A100 / H100 do NOT (they lack RT cores and won't initialize/render). "Bigger GPU" is the wrong instinct here.

How they fit together (the flow)

  1. A GPU (L40S) runs the Docker container.
  2. Inside, Isaac Sim simulates physics; Isaac Lab wraps it as an RL task.
  3. RSL-RL runs PPO to improve the policy against our rewards.
  4. Checkpoints go to Hugging Face; the job runs on Modal or Lightning.
Interview answer — "what's your stack?"
"NVIDIA Isaac Lab on Isaac Sim for massively-parallel simulation, RSL-RL for PPO, the Unitree G1 asset, AMP for motion style, and GR00T for the foundation-model track. Everything's containerized in one Docker image so I can run the same setup on Modal serverless GPUs or a Lightning box — both L40S, because Isaac Sim needs RT cores."
