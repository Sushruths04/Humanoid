The next-steps ladder (locomotion → real VLA)

The key insight: the roadmap assumed "language works", but the code fed a constant command — so making language informative is the real unlock, and it comes before vision, not after.

Wakeboarding track

  1. GPU smoke (Day-1 checklist) → fix runtime glue.
  2. Stage 1 discovery with the pull-speed curriculum (→ 30 km/h).
  3. Stage 2 deployable + AMP + domain randomization.
  4. Eval sweep + ablations + demo video.
  5. Phase 2: the webcam coaching loop.

Definition of "done" (for the portfolio)

  • Each task: a saved checkpoint + log + eval JSON + a short video, not just prose.
  • A results table with success / fall / episode-length per task.
  • At least one honest ablation (language on/off; vision vs state).
