CHAPTER 12

Glossary

Every bit of jargon in this site, in one plain sentence.

TermIn plain English
Reinforcement Learning (RL)Learning by trial and error, guided by rewards.
PolicyThe robot's "brain" — a neural network that maps what it sees to what it does.
PPOThe popular, stable RL algorithm that improves the policy in small safe steps.
RewardA number each step telling the robot how well it's doing.
Reward hackingWhen the robot games the reward instead of doing the real task.
AblationTurning off one ingredient to measure how much it mattered.
Isaac SimNVIDIA's GPU physics simulator.
Isaac LabThe RL toolkit built on Isaac Sim.
RSL-RLA fast PPO trainer for legged robots.
Unitree G1The real humanoid robot we train.
Domain randomizationRandomizing the sim's physics so the policy survives the real world.
Sim-to-realMaking something trained in simulation work on real hardware.
EpisodeOne attempt; it ends on success, timeout, or a fall.
UprightnessHow vertical the torso is (1 = perfectly upright).
CurriculumStarting easy and ramping difficulty as the robot improves.
VLAVision-Language-Action: a robot that sees, understands instructions, and acts.
GR00TNVIDIA's pre-trained humanoid "foundation model" we fine-tune.
Foundation modelA big pre-trained model you adapt to your task with little data.
AMPA reward that makes motion look like real reference clips.
HumanUPThe two-stage method for teaching a humanoid to get up.
EmbeddingTurning a word/instruction into a list of numbers a network can use.
CNNA neural network that processes images.
Domain (the "5 rules")The real wakeboarding technique we encode into rewards.
DockerA portable box containing the whole software setup.
Modal / Lightning AICloud GPU services (serverless / persistent).
RT coresGPU hardware Isaac Sim needs (L40S/L4 have them; A100/H100 don't).
